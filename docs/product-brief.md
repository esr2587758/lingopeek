# Lingobar 产品简报

## 一句话定位

Lingobar 是一个面向英语阅读、写作和记忆的即时语言工具栏。

它不是万能 productivity bar，而是 selection-first 的语言理解层：有选区时理解当前文本，无选区时接收输入，然后在原地给出翻译、拆解、收录和扩展。

## 核心承诺

任何文字，按一下，就进入学习、理解、表达的下一步。

## 产品标语

选中即解析，输入即生成。

英文方向：

Language command bar for reading, writing, and remembering English.

## 目标用户

- 正在大量阅读英文内容，但经常被单词、短语、长句打断的人。
- 想把遇到的表达沉淀成自己的短句库的人。
- 需要把中文想法快速变成自然英文的人。
- 已经使用翻译工具，但希望多一步理解和积累的人。

## 关键区别

PopClip 更像选中文本后的快捷操作菜单。Lingobar 更像选中文本后的语言理解层。

它不只处理复制、搜索、翻译，而是把文本变成一个学习对象：

- 单词：释义、词根、搭配、例句、近义词、语境差异。
- 短语：结构拆解、惯用表达、可替换说法。
- 句子：从句关系、自然翻译、口语/书面改写。
- 段落：总结、重点词汇、表达积累。
- 用户输入：生成表达、纠错、扩写、翻译成英文。

## MVP 范围

第一版只做五件事：

1. 选中文本后，通过全局快捷键弹出靠近选区的 bar。
2. bar 提供翻译、拆解、收藏、例句、发音这些语言动作。
3. 点击动作后，在 bar 下方展开结果面板。
4. 无选区时，通过同一个快捷键弹出输入框，支持输入或粘贴文本并询问 AI。
5. 结果面板支持复制、保存、继续追问和插入当前 App。

## 暂不做

- 截图、日历、文件、脚本等泛 productivity 功能。
- 复杂知识库和多端同步。
- 大而全的 AI agent 面板。
- 过早的平台生态和插件市场。

## 首屏交互模型

### 有选区

1. 用户在任意 App 选中一句话、短语或单词。
2. 按全局快捷键。
3. 屏幕上靠近选区弹出工具条。
4. 用户选择翻译、拆解、收藏、例句或发音。
5. 工具条下方展开结果。
6. 用户复制、保存、继续展开，或插入到当前 App。

### 无选区

1. 用户按全局快捷键。
2. 屏幕中央或光标附近弹出输入框。
3. 用户输入、粘贴或语音输入。
4. 回车后生成翻译、表达建议、纠错或扩写。
5. 用户继续追问、保存、复制或插入当前 App。

## 命名判断

当前建议优先使用 Lingobar。

原因：

- PhraseBar 更偏短语学习，边界清楚，但未来扩展到写作、多语言和 AI 表达时略窄。
- Lingobar 更像语言工具层，适合承载阅读、写作、记忆和多语言扩展。
- LingoPeek 作为仓库名也可以保留，产品名可以测试 Lingobar / LingoPeek 两条路线。

## 成功标准

早期版本不追求功能多，而追求一次使用后形成肌肉记忆：

- 用户能在 2 秒内从选中文本到看见翻译或拆解。
- 用户能把一个好表达一键存入本地短句库。
- 用户能用同一个入口完成“读懂”和“写出来”。
- 用户不会感觉自己打开了一个聊天机器人，而是感觉文本本身多了一层语言解释能力。
