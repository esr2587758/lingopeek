# Lingobar 产品简报

## 一句话定位

Lingobar 是一个面向英语阅读、写作和记忆的即时语言工具栏。

它不是万能 productivity bar，而是 selection-first 的语言理解层：有选区时理解当前文本，无选区时接收输入，然后在原地给出翻译、语法、收藏、例句、发音和自然英文改写。

## 核心承诺

任何文字，按一下，就进入学习、理解、表达的下一步。

## 产品标语

选中即理解，输入即改写。

英文方向：

Language command bar for reading, writing, and remembering English.

## 目标用户

- 正在大量阅读英文内容，但经常被单词、短语、长句打断的人。
- 想把遇到的表达沉淀成自己的短句库的人。
- 需要把中文想法快速变成自然英文的人。
- 已经使用翻译工具，但希望多一步理解和积累的人。

## 关键区别

PopClip 更像选中文本后的快捷操作菜单。Lingobar 更像选中文本后的语言理解层。

它不只处理复制、搜索、翻译，而是把文本变成一个学习对象：

- 单词：释义、词根、搭配、例句、近义词、语境差异。
- 短语：结构拆解、惯用表达、可替换说法。
- 句子：从句关系、自然翻译、口语/书面改写。
- 段落：总结、重点词汇、表达积累。
- 用户输入：把中文、英文、中英混合或粗糙想法改写成自然英文。

## MVP 范围

第一版只做五件事：

1. 选中文本后，通过全局快捷键弹出靠近选区的 bar。
2. bar 提供翻译、语法、改写、例句、收藏、发音这些语言动作。
3. 点击动作后，在 bar 下方展开结果面板。
4. 无选区时，通过同一个快捷键弹出输入框，支持输入或粘贴文本并改写成自然英文。
5. 结果面板支持复制、收藏和上下文更多动作；第一版写回当前 App 先通过复制完成。

## MVP 交付标准

这一版完成的含义是：用户能在本机长期自用的 macOS app，而不是面向多人分发的正式发布包。

必须完成：

- 全局快捷键唤醒和关闭 Lingobar。
- 有选区时进入选区模式，无选区时进入输入模式。
- OpenAI-compatible AI token / base URL / model 配置可用，所有语言结果依赖 AI 生成。
- Accessibility 权限引导可用，选区读取能力稳定。
- 支持复制、收藏、token 设置和稳定的关闭 / 再次唤醒行为。

暂不要求：

- 签名、公证、自动更新。
- 正式安装包和分发页面。
- 多用户账号、多端同步或团队协作功能。

## AI 依赖

Lingobar MVP 不采用本地优先或本地兜底的语言结果策略。所有翻译、语法、改写、例句和发音说明都依赖 AI 请求。

必须先完成 AI token 设置，才能使用 Lingobar 的语言功能。如果没有配置 token，首次启动或触发 Lingobar 时应进入设置引导 / AI 设置状态，而不是展示正常工具栏或伪造的本地结果。可保留开发期 fixture 或测试数据，但它不属于面向用户的结果路径。

AI 设置不应在产品语言上绑定到单一供应商。MVP 使用 OpenAI-compatible 配置模型：API token、base URL、model；默认值可以指向 DeepSeek，但设置区标题和领域语言应使用 `AI 设置`。

API token、Base URL 和 model 存储在用户偏好设置中，MVP 不依赖 macOS Keychain，避免开发期和自用场景反复触发 Keychain 授权弹窗。设置页可以显示 token 是否已配置，并提供粘贴新 token、清除 token 的操作，但不回显完整 token。

## 首次设置门禁

Lingobar MVP 需要先完成设置门禁，才能进入正常选区模式或输入模式。

设置门禁包含：

- AI access：已配置 API token、base URL 和 model。
- Accessibility permission：已授权 macOS Accessibility 权限，用于稳定读取选区和触发复制回退。

未完成设置时，触发 Lingobar 应显示设置引导，而不是正常语言动作。设置完成后，再次触发进入正常工作流。

MVP 不做流式输出。用户触发语言动作后显示 loading，等待完整 AI 响应返回后一次性渲染结果。

AI 响应必须使用结构化 JSON，而不是让 UI 从自由文本中猜测内容。App 按语言动作解析为固定结果模型，例如标题、主结果、辅助条目、默认收藏项和上下文更多动作。JSON 解析失败时显示错误并允许重试。

## 结果与动作规则

AI 结果使用统一结构：`title`、`summary`、`rows`、`chips`、`moreActionTitle`、`defaultCollectionItem`。UI 按当前语言动作渲染固定面板，不把 AI 响应当作聊天文本直接显示。

各语言动作的结果规则：

- `翻译`：中文自然翻译 + 关键表达 + 语感说明；上下文更多动作是 `解释更多`。
- `语法`：只支持英文；中文或混合内容时保留按钮位置但置灰。结果包含主干、修饰、逻辑和可收藏句型；上下文更多动作是 `继续拆解`。
- `改写`：一个主版本 + 口语、正式、简洁、地道等变体；上下文更多动作是 `更多版本`。
- `例句`：返回 3-5 个可复用英文例句，不做大段解释；上下文更多动作是 `更多例句`。
- `发音`：MVP 不做录音评分。先使用系统朗读 / 播放，并由 AI 返回重音、连读和易错点；上下文更多动作是 `慢速播放`。

错误规则：

- AI 请求失败、token 无效、JSON 解析失败都显示错误面板。
- 错误面板提供 `重试` 和 `打开 AI 设置`。
- 不降级展示本地假结果。

收藏与复制规则：

- `收藏` 收藏当前面板的默认收藏项，存入本地收藏数据。
- MVP 只做保存和最近收藏展示，不做搜索、标签、多端同步。
- `复制` 复制当前主结果。
- MVP 不做直接插入当前 App；写回当前 App 留到后续版本。

快捷键和关闭规则：

- MVP 快捷键固定为 `⌥⌘L`。
- Lingobar 已打开时再次触发：当前选区没变则关闭；有新选区则重新读取并刷新；无选区则聚焦输入框。
- `Esc` 关闭 Lingobar。
- 有草稿或结果时，点击外部不关闭。

语音输入规则：

- 语音输入不进入 MVP。
- 输入态可以保留麦克风图标作为未来能力占位，但点击时只显示 `暂未启用` 或保持禁用。
- MVP 不申请麦克风权限或语音识别权限。

## 暂不做

- 截图、日历、文件、脚本等泛 productivity 功能。
- 语音输入、录音评分和语音识别工作流。
- 复杂知识库和多端同步。
- 大而全的 AI agent 面板。
- 过早的平台生态和插件市场。

## 首屏交互模型

### 有选区

1. 用户在任意 App 选中一句话、短语或单词。
2. 按全局快捷键。
3. 屏幕上靠近选区弹出工具条。
4. 用户选择翻译、语法、改写、例句、收藏或发音。
5. 工具条下方展开结果。
6. 用户复制、收藏，或使用当前动作对应的更多操作。

### 无选区

1. 用户按全局快捷键。
2. 屏幕中央或上次拖动位置弹出输入框。
3. 用户输入或粘贴文本。
4. 回车后得到自然英文改写结果。
5. 用户复制、收藏，或请求更多版本。

## 命名判断

当前建议优先使用 Lingobar。

原因：

- PhraseBar 更偏短语学习，边界清楚，但未来扩展到写作、多语言和 AI 表达时略窄。
- Lingobar 更像语言工具层，适合承载阅读、写作、记忆和多语言扩展。
- LingoPeek 作为仓库名也可以保留，产品名可以测试 Lingobar / LingoPeek 两条路线。

## 成功标准

早期版本不追求功能多，而追求一次使用后形成肌肉记忆：

- 用户能在 2 秒内从选中文本到看见翻译或语法结果。
- 用户能把一个好表达一键存入本地短句库。
- 用户能用同一个入口完成“读懂”和“写出来”。
- 用户不会感觉自己打开了一个聊天机器人，而是感觉文本本身多了一层语言解释能力。
